# LOKACIA.KZ — Стратегия платежей и эквайринга

**Дата:** 23 апреля 2026
**Для:** Бухгалтер
**От:** Канат Рымтайев

---

## Задача

Настроить приём платежей на маркетплейсе lokacia.kz так, чтобы:
1. Деньги автоматически делились: комиссия платформы → на наш счёт, остальное → на счёт хоста
2. На наш счёт приходила **только комиссия** (не вся сумма), чтобы не было налоговой нагрузки на полный оборот
3. Арендаторы могли платить **любой картой** (Visa/Mastercard любого банка) + **Kaspi Pay**

---

## Как устроено у конкурентов (Giggster, Peerspace — USA)

- Используют **Stripe Connect** — деньги автоматически split'ятся
- Платформа выступает **агентом** (agent model), а не принципалом
- Доход платформы = только комиссия (~15%)
- Хост получает выплату через 1-3 дня после мероприятия
- **Stripe не работает в Казахстане** — нужна альтернатива

## Рекомендуемые провайдеры для Казахстана

### 1. PayBox.money — ОСНОВНОЙ провайдер

| Параметр | Значение |
|----------|---------|
| **Сайт** | paybox.money |
| **Split-платежи** | Да, маркетплейс API |
| **Escrow** | Да (заморозка до завершения бронирования) |
| **Карты** | Visa, Mastercard (любой банк КЗ и мира) |
| **Комиссия** | 2.5-3.5% от суммы транзакции |
| **Особенность** | Автоматическое разделение: комиссия → наш счёт, остальное → счёт хоста |

**Почему PayBox:**
- Единственный крупный провайдер в КЗ с нативным маркетплейс-split
- На наш счёт приходит только комиссия → налог считается только с неё
- Escrow защищает и хоста и арендатора

**Что нужно от бухгалтера:**
- Подать заявку на маркетплейс-тариф (paybox.money → "Для маркетплейсов")
- Предоставить документы ТОО
- Уточнить точную комиссию для нашего объёма

### 2. Kaspi Pay — ДОПОЛНИТЕЛЬНЫЙ метод

| Параметр | Значение |
|----------|---------|
| **Split-платежи** | Нет |
| **Комиссия** | 0-1% |
| **Популярность** | ~80% населения КЗ |

**Проблема:** Kaspi Pay не поддерживает автоматический split. Вся сумма приходит на наш счёт, затем мы вручную переводим хосту.

**Решение:** Агентский договор с каждым хостом (см. ниже).

### 3. Альтернативы (на рассмотрение)

| Провайдер | Split | Комиссия | Комментарий |
|-----------|:---:|---------|------------|
| **Wooppay** | Да | 2-3% | Казахстанский, есть маркетплейс-решение |
| **CloudPayments** | Да | 2.8-3.5% | Через партнёра в КЗ |
| **Freedom Pay** | Нет | 2-3% | Есть API, но split вручную |
| **Halyk ecom** | Нет | 1.8-2.5% | Интернет-эквайринг без split |

## Налоговая схема

### Вариант A: PayBox split (рекомендуемый)

```
Арендатор платит 100,000 тг
    │
    └── PayBox автоматически делит:
        → 85,000 тг → напрямую на счёт хоста
        → 15,000 тг → на счёт Lokacia (наша комиссия)

Наш доход = 15,000 тг
Налог (упрощёнка 3%) = 450 тг
```

### Вариант B: Kaspi Pay без split

```
Арендатор платит 100,000 тг
    │
    └── Вся сумма → на счёт Lokacia
        → Мы переводим 85,000 хосту
        → По агентскому договору наш доход = 15,000

Наш доход = 15,000 тг (по договору)
Налог (упрощёнка 3%) = 450 тг
```

**Без агентского договора:** весь оборот (100,000) считается доходом → налог 3,000 тг вместо 450 тг.

## Агентский договор

**Зачем:** Юридически закрепить что Lokacia — агент (посредник), а не продавец. Доход = только комиссия.

**Основание:** Гражданский Кодекс РК, Глава 43 "Агентский договор"

**Что должен содержать:**
1. Lokacia (ТОО) = агент
2. Хост = принципал
3. Предмет: агент оказывает услуги по поиску арендаторов для площадок принципала
4. Вознаграждение агента: 15% от суммы каждого бронирования
5. Остальные 85% принадлежат принципалу и перечисляются ему
6. Порядок расчётов: через электронную платформу lokacia.kz

**Формат:** Оферта на сайте (принимается при регистрации хоста) — не нужна бумажная подпись.

**Что нужно от бухгалтера:**
- Подготовить или проверить шаблон агентского договора
- Убедиться что он соответствует требованиям КГД (комитет гос. доходов)
- Настроить учёт: агентское вознаграждение = доход, транзитные суммы = не доход

---

## Юридическая структура

| Параметр | Значение |
|----------|---------|
| **Форма** | ТОО |
| **Налоговый режим** | Упрощённая декларация (3% от дохода) |
| **Доход** | Только комиссия (агентское вознаграждение) |
| **Лимит упрощёнки** | До 144 МЗП (~12,000,000 тг/год) |
| **НДС** | Не плательщик (до порога) |

## QR-платежи

| QR | Split | Комментарий |
|----|:---:|------------|
| Kaspi QR | Нет | Один получатель, split невозможен |
| Halyk QR | Нет | Аналогично |
| Freedom QR | Нет | Привязан к эквайрингу Freedom |

**Вывод:** QR-платежи в КЗ не поддерживают автоматический split. Можно принимать через QR на счёт платформы + агентский договор.
