# Правила разработки (TDD)
В этом проекте СТРОГО используется Test-Driven Development (TDD). 
При добавлении нового функционала или изменении старого, ты (ИИ) ОБЯЗАН:
1. Сначала написать падающий тест (Red).
2. Только после этого писать минимальный код реализации, чтобы тест прошел (Green).
3. Провести рефакторинг (Refactor).
Никогда не пиши логику до написания тестов.
